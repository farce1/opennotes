**Part B: Startup Shortcut from Store (lib.rs)**

Currently the shortcut is hardcoded as `"cmdorcontrol+shift+r"` in the global-shortcut plugin builder. Update lib.rs to read the user's saved shortcut from the Tauri store file at startup, BEFORE the builder is constructed.

**Approach: Direct file read before builder construction.** The Tauri store plugin v2 saves to `{AppData}/settings.json`. The app's bundle identifier is `com.opennotes.app` (from `src-tauri/tauri.conf.json`), so the resolved store path on macOS is:

```
~/Library/Application Support/com.opennotes.app/settings.json
```

Add a helper function and call it before the builder:

```rust
fn read_stored_shortcut() -> Option<String> {
    let home = std::env::var("HOME").ok()?;
    let store_path = std::path::PathBuf::from(&home)
        .join("Library/Application Support/com.opennotes.app/settings.json");
    let contents = std::fs::read_to_string(&store_path).ok()?;
    let parsed: serde_json::Value = serde_json::from_str(&contents).ok()?;
    parsed.get("recordingShortcut")
        .and_then(|v| v.as_str())
        .map(|s| {
            // The store saves "CommandOrControl+Shift+R" format
            // Convert to Tauri shortcut format: lowercase
            s.to_lowercase().replace("commandorcontrol", "cmdorcontrol")
        })
}
```

Then at the top of `run()`, before the builder:

```rust
pub fn run() {
    // ... migrations vec ...

    let startup_shortcut = read_stored_shortcut()
        .unwrap_or_else(|| "cmdorcontrol+shift+r".to_string());

    // ... recording_state, transcription_state, builder setup ...

    #[cfg(desktop)]
    {
        builder = builder.plugin(
            tauri_plugin_global_shortcut::Builder::new()
                .with_shortcuts([startup_shortcut.as_str()])
                .expect("Failed to register shortcut")
                .with_handler(|app, _shortcut, event| {
                    if event.state == ShortcutState::Pressed {
                        let _ = app.emit("recording-toggle", ());
                    }
                })
                .build(),
        );
    }
    // ...
}
```

This reads the settings file synchronously before any Tauri initialization. If the file doesn't exist (first launch) or is unreadable, it falls back to the default shortcut. No async, no store plugin dependency at this point.
