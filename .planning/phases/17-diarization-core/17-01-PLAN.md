**decode.rs** — Create `src-tauri/src/diarization/decode.rs` implementing OGG/Opus decode:

```rust
pub fn decode_ogg_opus_to_f32(path: &std::path::Path) -> Result<Vec<f32>, String>
```

Implementation:
1. Open the file at `path` with `std::fs::File::open`.
2. Create `ogg::reading::PacketReader::new(file)`.
3. Create `opus::Decoder::new(48_000, opus::Channels::Mono)`.
4. Iterate packets with `reader.read_packet()`. For each packet:
   - Skip header packets: if `packet.data.starts_with(b"OpusHead")` or `packet.data.starts_with(b"OpusTags")`, continue. But FIRST, when encountering the `OpusHead` packet, parse the pre-skip value from bytes 10-11 (little-endian u16) and store it.
   - For audio data packets, call `decoder.decode_float(&packet.data, &mut buf, false)` where `buf` is `vec![0.0f32; 5760]` (max 120ms at 48kHz).
   - Extend `samples` with `&buf[..n]` where `n` is the decode result.
5. After all packets decoded, discard the first `pre_skip` samples from the output buffer (these are encoder priming samples that would corrupt speaker embeddings).
6. Return the f32 samples at 48kHz sample rate.

**IMPORTANT:** Handle the `PacketReader::read_packet()` API correctly. The method signature in ogg 0.9 returns `Result<Option<Packet>, OggReadError>` — loop with `while let Some(packet) = reader.read_packet().transpose() { match packet { Ok(pkt) => ..., Err(e) => eprintln and continue } }`. If the API is different, adapt accordingly — check the actual return type at compile time.

**CRITICAL on pre-skip:** If pre-skip is not handled, the first few hundred milliseconds of decoded audio are garbage and produce incorrect speaker embeddings for the start of the recording. Parse from OpusHead header, discard those samples.

**Fallback:** If `ogg` + `opus` decode fails to compile or produces incorrect results, implement a fallback using `std::process::Command::new("ffmpeg")` with args `["-i", path, "-f", "f32le", "-ar", "48000", "-ac", "1", "pipe:1"]` and read stdout bytes as f32 samples. But try the ogg+opus approach first.

**worker.rs** — Create `src-tauri/src/diarization/worker.rs` with:

```rust
pub fn run_worker(
    pool: SqlitePool,
    data_dir: PathBuf,
    meeting_id: i64,
    on_event: Channel<DiarizationEvent>,
) -> Result<(), String>
```

This function runs synchronously on a `std::thread::spawn` thread (called from mod.rs). Steps:

1. **Set diarization_status to 'running'**: Use a new Tokio runtime (`tokio::runtime::Builder::new_current_thread().enable_all().build()`) inside the thread to execute the async SQLite query: `UPDATE meetings SET diarization_status = 'running' WHERE id = ?`. This is needed because sqlx is async-only. Build a small current-thread runtime at the top of the worker for all DB operations.

2. **Load audio path**: Query `SELECT audio_path FROM meetings WHERE id = ?` to get the recording file path.

3. **Decode OGG/Opus**: Call `decode::decode_ogg_opus_to_f32(&audio_path)` to get f32 samples at 48kHz.

4. **Resample to 16kHz**: Use `rubato::FftFixedIn::<f32>::new(48000, 16000, chunk_size, 2, 1)` (mono). Process the 48kHz samples through the resampler to produce 16kHz samples. Follow the same resampling pattern used in `src-tauri/src/transcription/resampler.rs`.

5. **Run diarization**: Initialize `sherpa_rs::diarize::Diarize::new(seg_model, emb_model, config)` with:
   - `DiarizeConfig { num_clusters: None, threshold: Some(0.5), min_duration_on: Some(0.3), min_duration_off: Some(0.5), provider: Some("cpu".to_string()), debug: false }`
   - Progress callback that sends `on_event.send(DiarizationEvent::Progress { percent })` — the callback receives `(computed: i32, total: i32)` and returns `0` to continue.
   - Call `diarizer.compute(samples_16k, Some(Box::new(progress_cb)))`.

6. **Align segments to transcripts**: Query all transcript rows for this meeting: `SELECT id, start_time_ms, (start_time_ms + 1000) as end_time_ms FROM transcripts WHERE meeting_id = ? ORDER BY segment_index`. For each transcript row, find the diarization segment with the largest time overlap (midpoint-overlap matching per Research Pattern 4). Convert diarization `Segment.start` and `Segment.end` from seconds to milliseconds by multiplying by 1000.0.

   **CRITICAL:** Treat transcript `end_time_ms` as `start_time_ms + 1000` (placeholder value set in the forwarder). Do NOT assume stored `end_time_ms` is accurate.

7. **Write results to DB** (all in a transaction):
   - Delete any existing diarization data for this meeting: `DELETE FROM speaker_turns WHERE meeting_id = ?`, then update `transcripts SET speaker_id = NULL WHERE meeting_id = ?`, then `DELETE FROM speakers WHERE meeting_id = ?`.
   - Insert speaker_turns: one row per diarization segment with `meeting_id, speaker_index, start_ms, end_ms`.
   - Insert speakers: one row per unique speaker_index with `meeting_id, speaker_index, display_name = '', color_index = speaker_index`.
   - Update transcripts: set `speaker_id` to the corresponding speakers row ID based on the alignment.
   - Update meetings: `SET diarization_status = 'complete'`.

8. **Send completion event**: `on_event.send(DiarizationEvent::Complete)`.

9. **On error at any step**: Send `DiarizationEvent::Error { message }` and update `meetings SET diarization_status = 'failed' WHERE id = ?`.
