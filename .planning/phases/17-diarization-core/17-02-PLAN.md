**hooks/useDiarization.ts** — Create a new hook mirroring useSummary.ts patterns:

```typescript
export function useDiarization(meetingId: number | null)
```

State:
- `status: DiarizationStatus` — 'idle' | 'running' | 'complete' | 'error'
- `percent: number` — progress percentage (0-100)
- `speakers: SpeakerRow[]` — speaker list with display names
- `speakerTurns: SpeakerTurnRow[]` — raw turn segments
- `errorMessage: string | null`

Functions:
- `startDiarization()` — calls `invoke('start_diarization', { meetingId, onEvent: new Channel<DiarizationEvent>(...) })`. The Channel callback updates `percent` on progress, sets `status = 'complete'` and calls `loadDiarizationData()` on complete, sets `status = 'error'` on error.
- `renameSpeaker(speakerId: number, name: string)` — calls `invoke('rename_speaker', { speakerId, displayName: name })` then updates the local `speakers` state optimistically (no refetch needed — update the matching speaker in-place).
- `loadDiarizationData()` — calls `invoke('get_diarization_data', { meetingId })` and sets speakers + speakerTurns + status from the response. Called on mount (to load existing data) and after diarization completes.

useEffect on mount: if `meetingId` is set, call `loadDiarizationData()` to check for existing diarization results. If `diarizationStatus === 'complete'`, set `status = 'complete'`.

Return: `{ status, percent, speakers, speakerTurns, errorMessage, startDiarization, renameSpeaker, loadDiarizationData }`

**SpeakerTranscript.tsx** — Chat-style two-column transcript layout.

Props:
```typescript
interface SpeakerTranscriptProps {
  segments: TranscriptSegment[];
  speakers: SpeakerRow[];
  speakerTurns: SpeakerTurnRow[];
  transcriptRows: TranscriptRow[];  // needed for speaker_id mapping
  onRenameSpeaker: (speakerId: number, name: string) => void;
}
```

Implementation:
1. Build a `speakerMap: Map<number, SpeakerRow>` from `speakers` array (keyed by `speaker_id` from the `speakers.id` field).
2. Group consecutive segments by speaker_id (per Research Pattern 6 — `groupBySpeaker`):
   - Iterate `transcriptRows` in order. For each row, look up `row.speaker_id` in the speaker map.
   - If the current row has the same speaker_id as the previous, add to the current group. Otherwise, start a new group.
   - Each group: `{ speakerId: number, speakerRow: SpeakerRow | null, segments: TranscriptSegment[] }`
3. Render as a chat-style layout:
   - Each group is a "bubble block" with a left column showing the speaker label (name + color indicator) and a right column showing the grouped text segments.
   - Speaker label uses the display_name from `SpeakerRow`, falling back to "Speaker N" (1-indexed) if display_name is empty.
   - Speaker label is colored with `getSpeakerColor(speaker.color_index)`.
   - Each segment shows its timestamp (formatted with `formatElapsed`) and text.
   - Clicking the speaker label opens the `SpeakerPopover`.

Layout (Tailwind):
```
[Speaker label col ~120px] | [Segments col flex-1]
  Speaker 1 (blue dot)     |   00:05  Hello everyone
                           |   00:12  Let's get started
  Speaker 2 (red dot)      |   00:18  Sure, I have the agenda
                           |   00:25  First topic is...
```

Use `grid grid-cols-[auto_1fr]` or `flex` with a fixed-width left column. The speaker label appears once per group, vertically centered or at the top.
