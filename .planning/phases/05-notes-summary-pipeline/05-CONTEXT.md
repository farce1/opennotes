# Phase 5: Notes/Summary Pipeline - Context

**Gathered:** 2026-02-27
**Status:** Ready for planning

<domain>
## Phase Boundary

Transform completed transcripts into structured meeting notes using a local LLM. Includes Ollama auto-setup during onboarding, post-meeting summary generation with streaming output, inline editing, and export. The transcript never leaves the machine — local-only processing is a hard requirement.

</domain>

<decisions>
## Implementation Decisions

### Summary structure
- Full structured format with four sections: Overview, Key Points, Decisions Made, Action Items
- Overview is a detailed paragraph (5-8 sentences) covering topics discussed, participants mentioned, and conclusions
- Action items formatted as assignee + task + deadline (e.g., "@John: Review the proposal by Friday")
- Meeting title auto-generated by LLM from transcript content; user can edit after

### LLM provider
- Local-only — no cloud API option. Privacy is non-negotiable; transcripts must never leave the machine
- Ollama is the sole LLM provider
- Auto-install Ollama during onboarding if not already present, then pull a recommended model
- Onboarding flow: detect Ollama → if missing, guide install → pull model → ready
- This mirrors the existing ASR model download pattern from Phase 3 — same UX principle of zero-config for the user

### Generation trigger & timing
- Auto-generate summary when recording stops — no extra button needed
- User can re-generate at any time if unsatisfied with the result
- Summary streams token-by-token as the LLM generates it — user can start reading immediately
- Inline editing supported: user can fix errors, add notes, tweak wording after generation; saves automatically

### Long meeting handling
- Claude's Discretion — choose the best strategy for transcripts exceeding LLM context window (chunked summarization, hierarchical summarization, or similar)

### Summary display
- Tab layout: "Summary" tab and "Transcript" tab in the meeting view
- Summary tab is the default/active tab when a meeting finishes — user sees streaming summary immediately
- Transcript tab is one click away for reference
- Summary rendered as rich markdown: headings, bold text, bullet points, checkboxes for action items

### Export
- Three export formats: Markdown file (.md), clipboard copy (formatted), and PDF
- Export applies to the summary (not just raw transcript)

### Claude's Discretion
- Long meeting chunking/hierarchical summarization strategy
- Specific Ollama model recommendation (e.g., Llama 3.2 8B, Phi-4-mini, Qwen 2.5)
- Ollama auto-install mechanism per platform (homebrew on macOS, installer on Windows, package manager on Linux)
- Streaming implementation details (SSE, polling, etc.)
- PDF generation library/approach
- Summary prompt engineering and section formatting

</decisions>
