Create `src-tauri/src/audio/` directory with four files.

**mixer.rs** — Audio mixing utilities:
- `mix_to_stereo(mic: &[f32], system: &[f32], output: &mut Vec<f32>)` — Interleaves mic samples as left channel and system audio as right channel. If system audio is None/empty (macOS < 14.6 fallback), duplicate mic to both channels.
- `rms_level(samples: &[f32]) -> f32` — Compute RMS amplitude for level metering.
- `f32_to_i16(samples: &[f32]) -> Vec<i16>` — Convert f32 [-1.0, 1.0] to i16 with clamping: `(s.clamp(-1.0, 1.0) * i16::MAX as f32) as i16`.

**encoder.rs** — Opus encoding thread:
- `struct EncoderConfig { path: PathBuf, sample_rate: u32, channels: u16 }` — Config for encoder creation.
- `fn run_encoder(config: EncoderConfig, rx: mpsc::Receiver<Vec<i16>>)` — Encoder loop function designed to run in `std::thread::spawn`. Creates `OpusEncoder::create_file()` at 48kHz, 2 channels, MonoStereo mapping. Sets VBR mode and 24kbit/s per channel bitrate (48kbit/s total) for ~1MB/min speech. Loops on `rx.recv()`, calling `encoder.write(&samples, 1)`. On channel close (sender dropped), calls `encoder.drain()` to finalize the Ogg file.
- Handle errors gracefully — log but don't panic. If encoder creation fails, log error and return immediately.

**capture.rs** — cpal stream management:
- `fn build_mic_stream(tx: mpsc::SyncSender<Vec<f32>>, app: AppHandle) -> Result<cpal::Stream, String>` — Uses `cpal::default_host()`, gets default input device, requests 48kHz f32 mono config (fall back to device default if 48kHz unavailable — check `supported_input_configs()` first). Builds input stream. In the data callback: compute RMS via `mixer::rms_level`, emit `app.emit("audio-level", level)` throttled to ~20 times/sec (use a frame counter or `Instant` comparison — do NOT use Mutex or blocking calls in the callback), and `tx.try_send(data.to_vec())`. Returns the stream (caller must call `.play()`).
- `fn build_loopback_stream(tx: mpsc::SyncSender<Vec<f32>>) -> Option<cpal::Stream>` — macOS only (`#[cfg(target_os = "macos")]`). Performs runtime macOS version check (parse `kern.osproductversion` via `sysctl` or read `/System/Library/CoreServices/SystemVersion.plist`). If < 14.6, return None with a log message. Otherwise, enumerate cpal hosts for CoreAudio, find the loopback input device, build input stream at 48kHz f32, send samples via `tx.try_send()`. Return Some(stream) on success, None on any failure (log warnings). NOTE: The exact cpal 0.17 loopback API surface should be verified at implementation time — read cpal source if the expected device enumeration pattern doesn't match. If loopback cannot be initialized, this is NOT an error — the app falls back to mic-only.
- `fn macos_version() -> (u32, u32)` — Helper to get macOS major.minor version tuple.

**mod.rs** — Public API and recording state:
- `struct RecordingState` — Holds: `mic_stream: Option<cpal::Stream>`, `loopback_stream: Option<cpal::Stream>`, `encoder_tx: Option<mpsc::SyncSender<Vec<i16>>>`, `encoder_handle: Option<JoinHandle<()>>`, `is_paused: AtomicBool`, `is_recording: AtomicBool`. Wrap in `Arc<Mutex<RecordingState>>` stored as Tauri managed state.
- `pub fn start_recording(app: &AppHandle, output_path: PathBuf) -> Result<(), String>` — Creates sync_channel(128) for mic samples and separate channel for loopback samples. Spawns encoder thread with `encoder::run_encoder`. Builds mic stream and optional loopback stream. In the audio callbacks, use mixer to combine samples before sending to encoder. Stores streams and handles in RecordingState. Sets is_recording=true.
- `pub fn stop_recording(app: &AppHandle) -> Result<(), String>` — Drops mic_stream and loopback_stream (stops cpal). Drops encoder_tx (signals encoder thread to finish). Joins encoder_handle. Sets is_recording=false.
- `pub fn pause_recording(app: &AppHandle) -> Result<(), String>` — Sets is_paused=true. The audio callbacks check this flag and skip sending samples when paused (do NOT stop the cpal streams — just discard samples). Emit "recording-paused" event.
- `pub fn resume_recording(app: &AppHandle) -> Result<(), String>` — Sets is_paused=false. Emit "recording-resumed" event.

CRITICAL implementation notes:
- NEVER block in cpal callbacks (no mutex, no file I/O, no allocation beyond try_send)
- Use `try_send` (not `send`) in audio callbacks — dropped frames are acceptable
- Size sync_channel at 128 frames for burst tolerance
- The encoder thread is the ONLY thread doing disk I/O
- Explicitly request 48kHz sample rate from cpal. If the device doesn't support 48kHz, use the device's native rate AND initialize libopusenc at that same rate (do NOT mix rates)
- Do NOT call `default_output_device()` anywhere — it triggers unexpected mic permission on macOS (cpal issue #901)
